处理iPhone拍照的图片方向

  • imagick
  • php
  • 前端

在处理一张用户上传的图片时,发现在PC的chrome和iPhone的safrai显示方向不一样,以下将讲述为何同一张图片在不同浏览器上的显示方向不一样.

用手机或者其他数码产品拍照时,会以各种方向拿着手机,这样是如何保证在后来查看照片时,照片都以正确的方式显示在相册里?
现在手机中都会有方向传感器,在拍照时,它会将手机的旋转角度等信息存储在照片中,然后相机就可以按照这个角度来显示图片.
然而在不同的设备上,或者不同的浏览器上,对图片的旋转信息的处理方式都是不一样的,就好导致在不同设备或者不同浏览器上所看的图片方向不一致,出现偏差.
所以,我在用户将相片上传的时候,将图片旋转至正确的角度,再将处理完的图片上传.
代码如下:
//先获取图片的exif信息

$exif = exif_read_data($_tmp_name);
        $orientation = isset($exif['Orientation']) ? $exif['Orientation'] : null;
   // 如果存储了角度信息,则进行处理
        if (!empty($orientation)) {
            $this->image_fix_orientation($orientation,$_tmp_name);                
        }

function image_fix_orientation($orientation,$_tmp_name)
{
    //iPhone home键在右边时为正方像
    switch ($orientation) {
        //顺时针90°
        case 8:
            $rotateImage = 270;
            $imageOrientation = 1;
            break;
        //顺时针180°
        case 3:
            $rotateImage = 180;
            $imageOrientation = 1;
            break;
        //home键在下方,即正常握手机的方向
        case 6:
            $rotateImage = 90;
            $imageOrientation = 1;
            break;
    }
    if ($rotateImage) {
        if (class_exists('Imagick')) {
            $imagick = new \Imagick();
            $imagick->readImage($_tmp_name);
            $imagick->rotateImage(new \ImagickPixel(), $rotateImage);
            $imagick->setImageOrientation($imageOrientation);
            $imagick->writeImage($_tmp_name);
            $imagick->clear();
            $imagick->destroy();
        }
    }
}

imagick比PHP GD库有更多的功能和更高的销量,可以用它来进行图片的处理.
